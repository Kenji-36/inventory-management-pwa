## 2. 全体設計方針

### 2.1 デザインコンセプト
- **シンプル・クリーン**: 余計な装飾を排除し、コンテンツに集中
- **段階的な誘導**: 次に何をすべきか常に明確
- **視覚的フィードバック**: 操作に対する即座の反応
- **一貫性**: 全画面で統一されたUI/UX

### 2.2 レイアウト原則
- **1カラムレイアウト**: 集中力を保つため横幅を制限（最大800px）
- **固定ヘッダー**: プログレスバーを常時表示
- **十分な余白**: 読みやすさと操作性を確保
- **レスポンシブ**: デスクトップ優先だが、タブレットでも使用可能

### 2.3 カラーパレット（推奨）
- **プライマリカラー**: #4F46E5（インディゴ）- メインアクション
- **セカンダリカラー**: #10B981（グリーン）- 成功・完了
- **アクセントカラー**: #F59E0B（アンバー）- 警告・注意
- **エラーカラー**: #EF4444（レッド）- エラー
- **背景色**: #F9FAFB（ライトグレー）
- **テキスト**: #111827（ダークグレー）
- **ボーダー**: #E5E7EB（グレー）

### 2.4 タイポグラフィ
- **フォント**: システムフォント（-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif）
- **見出し**: 24-32px, Bold
- **本文**: 16px, Regular
- **補足テキスト**: 14px, Regular
- **行間**: 1.6-1.8

## 3. 共通コンポーネント

### 3.1 ヘッダー（全画面共通）

```
┌─────────────────────────────────────────────────────────┐
│  🎯 Blog Generator                                      │
│                                                         │
│  [1.テーマ] → [2.見出し] → [3.目次] → [4.本文] → [5.完成] │
│     ●            ○           ○          ○          ○    │
└─────────────────────────────────────────────────────────┘
```

**構成要素**:
- アプリケーション名（左上）
- プログレスバー（中央）
  - 5つのステップを横並び表示
  - 現在のステップ: 塗りつぶし円（●）
  - 未完了のステップ: 空円（○）
  - 完了したステップ: チェックマーク（✓）
  - ステップ間は矢印または線で接続

**サイズ**: 高さ80px、固定表示

---

### 3.2 ボタンスタイル

#### プライマリボタン
```
┌──────────────┐
│  次へ進む →  │  ← 背景: プライマリカラー、白文字
└──────────────┘
```
- 使用場面: メインアクション（次へ、生成、決定など）
- サイズ: 高さ48px、横幅auto（最小200px）
- ホバー: 色を10%暗く

#### セカンダリボタン
```
┌──────────────┐
│  ← 戻る      │  ← 背景: 白、枠線あり、テキスト色
└──────────────┘
```
- 使用場面: サブアクション（戻る、キャンセルなど）
- サイズ: 高さ48px、横幅auto
- ホバー: 背景を薄いグレーに

#### テキストボタン
```
 再生成 🔄
```
- 使用場面: 補助的なアクション
- サイズ: 高さ36px
- ホバー: 下線表示

## 4. 画面詳細設計

### 4.1 S-001: テーマ入力画面

#### 4.1.1 画面レイアウト

```
┌─────────────────────────────────────────────────────────┐
│  ヘッダー（プログレスバー: ステップ1が選択状態）          │
└─────────────────────────────────────────────────────────┘

              ┌─────────────────────────┐
              │                         │
              │   📝 記事のテーマを      │
              │      入力してください    │
              │                         │
              └─────────────────────────┘

              ┌─────────────────────────┐
              │ どんな記事を書きたいですか？│
              │                         │
              │ ┌─────────────────────┐ │
              │ │                     │ │
              │ │ [テキストエリア]     │ │
              │ │                     │ │
              │ │                     │ │
              │ │                     │ │
              │ └─────────────────────┘ │
              │                         │
              │ 💡 例: 「初心者向けの    │
              │    SEO対策の基本」       │
              │                         │
              │        0 / 500文字       │
              └─────────────────────────┘

              ┌──────────────────┐
              │  見出しを生成 →  │
              └──────────────────┘
```

#### 4.1.2 構成要素

| 要素 | 説明 | スタイル |
|------|------|----------|
| ページタイトル | 「記事のテーマを入力してください」 | 32px, Bold, 中央揃え |
| カードコンテナ | 入力フォームを囲む白背景のカード | 背景白、角丸8px、影あり |
| ラベル | 「どんな記事を書きたいですか？」 | 16px, Medium |
| テキストエリア | テーマ入力欄 | 高さ150px、幅100%、枠線 |
| プレースホルダー | 入力例を表示 | グレー文字 |
| ヒント | 💡 入力例 | 14px、薄い背景 |
| 文字数カウンター | 「0 / 500文字」 | 14px、右揃え |
| 生成ボタン | 「見出しを生成 →」 | プライマリボタン、中央配置 |

#### 4.1.3 バリデーション
- 空欄の場合: ボタン無効化（グレーアウト）
- 500文字超過: 赤文字で警告、ボタン無効化

#### 4.1.4 インタラクション
1. ユーザーがテキスト入力
2. リアルタイムで文字数カウント更新
3. 1文字以上入力でボタン有効化
4. ボタンクリックで次画面へ遷移

- ラジオボタン: 塗りつぶし

- カード: 影を強調、カーソルをポインターに

1. カードクリックで選択（ラジオボタンも連動）
2. 選択すると次へボタンが有効化
3. 再生成ボタンで新しい候補を生成（ローディング表示）
4. 戻るボタンでテーマ入力画面へ

#### 4.5.2 構成要素

| 要素 | 説明 | スタイル |
|------|------|----------|
| ページタイトル | 「記事が完成しました！」 | 32px, Bold, 中央揃え |
| サイドバー | 左側の操作パネル | 幅250px、固定 |
| プレビューエリア | 右側の記事表示 | 残りの幅、スクロール可能 |
| 編集ボタン | 編集モードに切り替え | サイドバー内 |
| プレビューボタン | プレビューモードに切り替え | サイドバー内、デフォルト選択 |
| エクスポートボタン | 各形式でダウンロード | サイドバー内、3つ |
| 編集に戻るボタン | 本文編集画面に戻る | サイドバー下部 |
| 新規作成ボタン | 新しい記事を作成 | サイドバー下部 |
| 記事タイトル | 見出しを大きく表示 | 36px, Bold |
| 目次表示 | 記事の目次 | リスト形式、リンク付き |
| 本文表示 | Markdownレンダリング | 16px、適切な行間 |

#### 4.5.3 プレビューモード

**表示スタイル**:
- 見出し: H2は28px、H3は22px
- 段落: 16px、行間1.8
- リスト: 適切なインデント
- 全体: ブログ記事風のスタイル

**スクロール動作**:
- 目次のリンククリックで該当セクションにスムーズスクロール
- 現在表示中のセクションを目次でハイライト

## 5. レスポンシブデザイン

### 5.1 ブレークポイント
- **デスクトップ**: 1024px以上（デフォルト）
- **タブレット**: 768px - 1023px
- **モバイル**: 767px以下（将来対応）

### 5.2 タブレット対応

**レイアウト調整**:
- メインコンテンツ幅: 最大700px
- カードの余白を縮小
- ボタンサイズを若干拡大（タップしやすく）
- フォントサイズは維持

**プレビュー画面**:
- サイドバーを上部に配置
- ボタンを横並びに

## 7. アニメーション・トランジション

### 7.1 画面遷移
- フェードイン: 300ms
- スライドイン: 400ms（左右）

### 7.2 要素の表示/非表示
- ローディング: スピナー回転
- アコーディオン: 200ms ease-out
- ホバー効果: 150ms

### 7.3 ドラッグ&ドロップ
- ドラッグ中: 不透明度50%
- ドロップ位置: 青い線で表示
- アニメーション: 250ms

## 8. エラー・エッジケース

### 8.1 API エラー
- タイムアウト: 30秒後にエラー表示
- レート制限: 「しばらく待ってから再試行」メッセージ
- 接続エラー: 「インターネット接続を確認」メッセージ

### 8.2 データ消失防止
- 自動保存: 各ステップ完了時
- ブラウザ閉じる前: 確認ダイアログ
- セッション復元: ページ再読み込み時

### 8.3 空状態
- 生成失敗: 「もう一度お試しください」
- 目次が空: 「項目を追加してください」
